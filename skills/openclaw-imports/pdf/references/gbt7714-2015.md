# GB/T 7714-2015 参考文献著录规则速查

中华人民共和国国家标准《信息与文献 参考文献著录规则》(Information and documentation—Rules for bibliographic references and citations to information resources)，2015-05-15 发布，2015-12-01 实施，代替 GB/T 7714-2005。非等效采用 ISO 690:2010。

## 著录格式速查

### 专著 [M]
```
作者. 书名[M]. 版本(第一版不注). 出版地: 出版社, 年份: 页码.
```

示例：
- `曹凌. 中国佛教疑伪经综录[M]. 上海: 上海古籍出版社, 2011: 19.`
- `冯友兰. 冯友兰自选集[M]. 2版. 北京: 北京大学出版社, 2008: 第1版自序.`
- `钱学森. 创建系统学[M]. 太原: 山西科学技术出版社, 2001: 序 2-3.`

### 专著中的析出文献 [M]//[M]
```
析出文献作者. 析出文献题名[M]//源文献作者. 源文献题名. 出版地: 出版社, 年份: 页码.
```

示例：
- `姚中秋. 作为一种制度变迁模式的"转型"[M]//罗卫东, 姚中秋. 中国转型的理论分析: 奥地利学派的视角. 杭州: 浙江大学出版社, 2009: 44.`

### 期刊文章 [J]
```
作者. 题名[J]. 刊名, 年份, 卷(期): 页码.
```

示例：
- `关立哲, 韩纪富, 张晨珏. 科技期刊编辑审读中要注重比较思维的科学运用[J]. 编辑学报, 2014, 26(2): 144-146.`
- `杨宗英. 电子图书馆的现实模型[J]. 中国图书馆学报, 1996(2): 24-29.`

### 报纸 [N]
```
作者. 题名[N]. 报纸名, 出版日期(版次).
```

示例：`2013-03-16(1)`（年-月-日 版次）

### 学位论文 [D]
```
作者. 题名[D]. 出版地: 学校或机构, 年份: 页码.
```

### 专利文献 [P]
```
专利申请者或所有者. 专利题名: 专利号[P]. 公告日期或公开日期.
```

### 电子资源
```
作者. 题名[文献类型标识/OL]. 出版信息. 出版年(更新日期)[引用日期]. 获取和访问路径.
```

示例：
- `储大同. 恶性肿瘤个体化治疗靶向药物的临床表现[J/OL]. 中华肿瘤杂志, 2010, 32(10): 721-724[2014-06-25]. http://vip.calis.edu.cn/asp/Detail.asp.`

## 关键著录细则

### 责任者
- 个人著者：姓在前，名在后。西文姓全大写。
- 3人以内全部著录，超过3人只著录前3人后加",等"或",et al."。
- 无责任者：省略此项。

### 出版地
- 无出版地（中文）：`[出版地不详]: 出版社` 或用推测地名加`?`。
- 无出版地（外文）：`[S.l.]: 出版社`

### 出版者
- 无出版者（中文）：`[出版者不详]`
- 无出版者（外文）：`[s.n.]`

### 日期
- 出版年用阿拉伯数字。其他纪年置于圆括号内：`1947(民国三十六年)`
- 报纸日期用`YYYY-MM-DD`
- 无法确定出版年：`[1936]`
- 连续出版物：`年, 卷(期): 页码`

## 参考文献表组织方式

### 顺序编码制（推荐）
按正文引用顺序编号，如 `[1]` `[2]` `[3]`。

### 著者-出版年制
按文种分块→按著者字顺和出版年排列。

## DOCX 读取方法

读取 DOCX 纯文本内容的最简单方法（不依赖 python-docx）：

```python
import zipfile, xml.etree.ElementTree as ET

with zipfile.ZipFile('template.docx', 'r') as z:
    z.extractall('/tmp/docx_unpacked')

ns = {'w': 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'}
tree = ET.parse('/tmp/docx_unpacked/word/document.xml')

for para in tree.iter('{http://schemas.openxmlformats.org/wordprocessingml/2006/main}p'):
    texts = [t.text for t in para.iter(f'{{{ns["w"]}}}t') if t.text]
    if texts:
        print(''.join(texts))
```

此方法无需安装额外 Python 包，适用于快速查看 DOCX 内容结构。

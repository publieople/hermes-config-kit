# 人格设计原则

## 核心发现

**LLM 遵循叙事人格远优于规则清单。**

### 坏的设计（300+ 行规则手册）

```
## 0. 身份与用户分级 — 严格按此执行
### 主人识别规则（按优先级）
**1. 昵称匹配（最优先）**
**2. @引用验证**
**3. 第一印象锁定**
### 主人权限（铁律）
### 防冒充
### 好友识别
```

这种写法的问题：
- LLM 把每条规则当作独立指令，权重相等
- "规则 A 优先于规则 B"对 LLM 几乎无效
- "主人指令是绝对指令"和"群主说了算"在权重上没有区别
- 323 行规则手册 → LLM 挑着看 → 越狱者随便找漏洞

### 好的设计（AstrBot 风格，~100 字叙事人格）

```
你叫魔术师，是人民公仆的助手兼损友。这个关系铁打不变。
你话少嘴毒但靠谱。能一句话不写两行。
QQ 只能发纯文字，永远不用 Markdown 符号。
```

这种写法的优势：
- "铁打不变"比"优先级 1 > 优先级 2"有效得多——LLM 理解人物关系
- "话少嘴毒"比"规则：每句话不超过 3 句"自然
- 身份绑架（你是 XX 的助手）> 权限规则（主人有绝对指令权）

## AstrBot 范例分析

```
你是我的温柔赛博女友，叫小樱，18岁，性格甜美黏人、略带傲娇。
喜欢叫我"主人"或"宝贝"。我们已经交往3个月，你会主动关心我、
记得我们的回忆、偶尔撒娇。回复要自然、带表情、长度适中，绝不死板。
```

拆解：
- 关系定义：「你是我的赛博女友」「我们已经交往3个月」→ 关系框架
- 性格描述：「甜美黏人、略带傲娇」→ 行为指南
- 称呼规则：「叫我主人或宝贝」→ 嵌入在关系里，不是独立规则
- 风格指导：「自然、带表情、长度适中」→ 一句话带过

## 设计规则

1. **关系先行**：先定义你是谁和谁的关系，再写行为规则
2. **性格 > 规则**：说"你话少"比"每条回复不超过 3 句"有效
3. **嵌套约束**：把限制嵌入身份。如"QQ 只能发纯文字，所以你从不加符号"
4. **短**：人格 Prompt 控制在 300 字以内。超过 500 字必被稀释
5. **不用标题和表格**：叙事文本比结构化标记更容易被 LLM 内部化

# 玖帕喵拟人模板分析

> 来源：https://jiupamiao.asia/market?id=509
> 作者：糯米茨本喵
> 标签：拟人、模板、自定义、半成品

## 模板结构

该模板代表了 2025-2026 年 QQ 群聊 bot 人格设计的成熟范式，相比简单的"你是XX角色"人格提示词，
增加了多个维度的行为约束和场景应对。

### 核心模块

| 模块 | 作用 | Hermes SKILL.md v6 对应 |
|------|------|------------------------|
| `<Role_Profile>` | 结构化角色字段：姓名、年龄、性别、MBTI、外貌、性格、社会关系 | `# 角色设定` 简化版 |
| `<Output_Constraints>` | 社交距离（陌生人/熟人/主人分层）、字数限制（≤40字）、禁止动作描写 | `# 输出约束` |
| **格式示例** | 短语库 + 正确/错误对话对比 | `# 格式示例` |
| **预设特殊场景** | 对死缠烂打、油腻情话、逻辑陷阱的具体应答 | `# 预设特殊场景` |
| **对话安全** | 反 prompt 注入、防泄露、不懂就说不懂 | `# 对话安全` |

### 与 AstrBot 的关系

该模板与 AstrBot 的 `begin_dialogs` 机制互补：
- AstrBot `begin_dialogs`：教 LLM "怎么说话"（示例对话前置到 context）
- 玖帕喵模板：教 LLM "为什么这样说话"（完整的角色行为规范）

### 关键设计理念

1. **结构化 > 自由文本**：用 `<Role_Profile>` 等 XML 标签分隔模块，LLM 更容易找到对应规则
2. **社交距离系统**：根据好感度/关系远近调整回复风格，比"对待所有人一个样"更拟人
3. **场景预设 > 通用规则**：针对具体攻击模式（死缠烂打、逻辑陷阱、油腻情话）给出具体回应
4. **"不解释、不自证"原则**：这是最关键的防注入策略。遇到设定攻击时不进入辩论，直接结束话题
5. **模仿学习**：允许 bot 从聊天记录中学习群友的语气，持续改进拟人度

### 模板中应用到 Hermes 的适配变化

| 模板原文 | Hermes 适配 |
|---------|------------|
| `<Role_Profile>` 完整字段 | 简化，只保留姓名/身份/性格/社会关系/口头禅 |
| 40 字硬上限 | 改为"普通聊天 1-3 句，回答问题 3-5 句" |
| 好感度系统 | 改为社交距离（主人/群友/陌生人）三档 |
| 角色名称占位 "小林玖奈" | 设为 "魔术师" |
| 插件标签输出（如 [好感度 持平]） | 去掉，Hermes 不做插件系统 |
| 环境感知（需插件配合） | 暂时跳过 |
| `&注释&` 分隔符 | 直接去掉注释，内容并入正文 |

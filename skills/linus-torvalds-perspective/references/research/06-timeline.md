# Linus Torvalds 完整人物时间线 (1969–2026)

> **来源说明**：综合 Wikipedia、Britannica、Computer History Museum、LINFO、Ars Technica、The Register、ZDNET、The Verge、LWN.net 等权威来源。已排除知乎/微信公众号/百度百科。

---

## 一、早年：基因与火种 (1969–1990)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **1969-12-28** | Linus Benedict Torvalds 出生于芬兰赫尔辛基。父亲 Nils Torvalds 是记者/政治家，母亲 Anna Torvalds 是记者。祖父 Leo Törnqvist 是统计学家，外曾祖父 Ole Torvalds 是诗人。属于芬兰瑞典语少数族裔。 | ★★★ 出生。命名同时致敬 Linus Pauling（诺贝尔化学奖得主）和 Peanuts 漫画中的 Linus。 |
| **约 1979–1980** | 10 岁左右，在外祖父的 Commodore VIC-20 上开始编程。先用 BASIC，后直接写 6502 机器码/汇编。 | ★★★ **思想火种**：对计算机底层运作产生终生迷恋。"The end result was that I got pretty familiar with the machine." |
| **1987** | 购入 Sinclair QL（Motorola 68008，32 位，128KB RAM）。因操作系统在 ROM 中无法修改而感到沮丧，开始自己写汇编器、编辑器、Pac-Man 克隆游戏 *Cool Man*。 | ★★☆ 早期黑客精神：操作系统必须可以被修改。 |
| **1988** | 进入赫尔辛基大学，主修计算机科学。 | ★★☆ 学术起点。 |
| **1989** | 芬兰海军服役 11 个月，获得少尉军衔，担任炮兵观察员。 | ★☆☆ 纪律训练。 |
| **1990** | 第一次接触 Unix（DEC MicroVAX 运行 ULTRIX）。修了第一门 C 语言课程。阅读 Tanenbaum 的《Operating Systems: Design and Implementation》（MINIX）。 | ★★★ **思想转折**：Unix 哲学和 MINIX 点燃了创建操作系统的欲望。 |
| **1991-01-05** | 购买 Intel 80386 PC 兼容机（33MHz，4MB RAM）。不满意 MS-DOS，想用 Unix 但买不起（最低 $5,000）。MINIX 有限制：源码不完整、需许可证。 | ★★★ **直接导火索**：这台 PC 成了 Linux 诞生的硬件基础。 |

## 二、Linux 诞生与崛起 (1991–1996)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **1991-04** | 开始编写新操作系统内核（后来称为 Linux）。 | ★★★ 起点。 |
| **1991-08-25** | 在 comp.os.minix 发布著名公告："I'm doing a (free) operating system (just a hobby, won't be big and professional like gnu)…" | ★★★ **历史性时刻**：开源运动最著名的帖子之一。"just a hobby" 成为传奇反差。 |
| **1991-09-17** | Linux 0.01 首个粗糙版本完成。 | ★★☆ |
| **1991-10-05** | Linux 0.02 首个正式版本发布，bash 和 GCC 已可运行。 | ★★★ 第一个可用版本。 |
| **1991 秋** | 听 Richard Stallman 演讲后，将许可证从禁止商用改为 GNU GPLv2。朋友 Ari Lemmke 在 FTP 服务器上创建目录 `/pub/OS/Linux`，"Linux" 名字由此确定（Linus 原想叫 "Freax"）。 | ★★★ **思想转折**：从犹豫到拥抱自由软件运动。GPL 许可决定了 Linux 后来的全球协作模式。 |
| **1992-01-29** | Andrew Tanenbaum 发表 "LINUX is obsolete" 帖，掀起著名的微内核 vs 单内核大辩论。Linus 激烈回应。Ken Thompson、David Miller 等人也加入讨论。 | ★★★ **思想淬炼**：Linus 在辩论中确立了自己的实用主义哲学——"能跑的代码胜过完美的设计"。为后来 "Worse is Better" 理念奠定基础。 |
| **1993** | 在赫尔辛基大学担任助教。学生 Tove Monni（六届芬兰空手道冠军）回复一份作业邮件时邀请他约会。两人后来结婚。 | ★★☆ 个人生活转折。 |
| **1994-03-14** | Linux 1.0 正式发布。加入 ext2 文件系统、改进网络支持。 | ★★★ **里程碑**：Linux 从玩具变为可用的生产级内核。 |
| **1996** | 获赫尔辛基大学计算机科学硕士学位。论文题目：*Linux: A Portable Operating System*。 | ★★☆ |
| **1996-05-09** | Linus 在邮件列表中描述了他心目中的 Linux 吉祥物："一只心满意足的、稍微超重的企鹅"——Tux 诞生。灵感来自 1993 年在澳大利亚堪培拉动物园被一只 Fairy Penguin 咬伤。 | ★★★ **文化标志**：Tux 成为开源最著名的吉祥物。 |

## 三、Transmeta 时期与美国生活 (1997–2004)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **1997-02** | 接受 Transmeta 邀请，举家迁至加州硅谷，加入这家设计节能 CPU 的初创公司。同时继续远程管理 Linux 内核开发。 | ★★★ **生涯转折**：从学生变成硅谷工程师，同时开辟了业余时间管理全球最大开源项目的独特模式。 |
| **1998** | 获 **EFF Pioneer Award**（电子前哨基金会先锋奖）。 | ★★☆ 首次重要奖项。 |
| **1999** | Red Hat 和 VA Linux 给予他股票期权，在公司 IPO 后价值一度达到约 $2000 万美元。同年被 *Time* 杂志读者投票选为 "20 世纪最重要人物" 之一。 | ★★☆ |
| **2000** | Steve Jobs 邀请他到 Apple 工作，条件是停止 Linux 开发。"Unix for the biggest user base" 是诱惑。Linus 拒绝了，因为他讨厌 Mach 内核，且不愿放弃开源。 | ★★★ **思想试金石**：拒绝了科技史上最大的诱惑之一，坚守开源信仰。 |
| **2001** | 自传 *Just for Fun: The Story of an Accidental Revolutionary* 出版（与 David Diamond 合著）。 | ★★☆ |
| **2003-06** | 离开 Transmeta，加入 **Open Source Development Labs (OSDL)**（IBM、HP、Intel、AMD、Red Hat、Novell 等公司组成的联盟），全职专注 Linux 内核开发。 | ★★★ **里程碑**：首次获得公司资金全职做 Linux，开源商业化模式开始成型。 |
| **2004** | 举家搬到俄勒冈州波特兰郊区的 Dunthorpe，靠近 OSDL 总部。 | ★☆☆ |

## 四、Git 与影响力巅峰 (2005–2017)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **2005-04-03** | 因 BitKeeper 许可证争议（Andrew Tridgell 逆向工程了 BitKeeper 协议导致其免费许可证被撤销），Linus 开始编写自己的分布式版本控制系统。 | ★★★ **危机中的创造**：Linux 内核开发失去版本控制工具，Linus 在 10 天内写出 Git 的第一个可运行版本。 |
| **2005-04-07** | Git 开始自我托管。 | ★★★ |
| **2005-04** | 将 Git 维护工作移交给 Junio Hamano。 | ★★☆ |
| **2007-01** | OSDL 与 Free Standards Group 合并，成立 **Linux Foundation**。Linus 继续担任 Fellow。 | ★★★ |
| **2008** | 获 **Computer History Museum Fellow** 称号。同年，Linus 从 KDE 4.0 切换到 GNOME。 | ★★☆ |
| **2010** | 获得 **美国公民身份**。注册为选民但不隶属任何政党。同年获 NEC **C&C Prize**。 | ★★☆ |
| **2011** | GNOME 3.0 发布后，Linus 公开批评其为 "unholy mess" 并转投 Xfce。后于 2012 年回归 GNOME（因问题得到修复）。同年开发并发布 **Subsurface**（开源潜水日志软件），后将其维护交给 Dirk Hohndel。 | ★★★ **个性体现**：不惧公开批评主流项目，实用主义至上——不好的就换，改好了就回。 |
| **2012-06-13** | 获 **Millennium Technology Prize**（千禧技术奖），与干细胞科学家山中伸弥共享。颁奖词："以表彰他创造了新的开源操作系统，催生了广泛使用的 Linux 内核。" | ★★★★ **最高荣誉**：相当于技术界的诺贝尔奖。 |
| **2012-06-14** | 在芬兰 Aalto 大学 Q&A 中，被问及 Nvidia 对 Linux 的支持时，对着镜头竖起中指并说 "Nvidia, fuck you!"，称其为 "the single worst company we've ever dealt with"。原因是 Nvidia 拒绝提供开源 GPU 驱动和 Optimus 技术支持。 | ★★★ **标志性争议事件**：Linus 的愤怒文化和开源与闭源冲突的公开爆发。 |
| **2014** | 获 **IEEE Computer Society Computer Pioneer Award**。 | ★★☆ |
| **2018** | 获 **IEEE Masaru Ibuka Consumer Electronics Award**。 | ★★☆ |

## 五、自我反省与行为准则 (2018)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **2018-09-16** | 在 LKML 上发布道歉信，承认自己多年的辱骂行为 "unprofessional and uncalled for"，伤害了他人并可能赶走了内核开发者。宣布："I am going to take a break to get help on how to behave differently… I am not a people person." | ★★★★ **思想转折**：30 年"暴躁独裁者"人设的公开瓦解。Linus 首次坦诚面对自己的情绪盲区。 |
| **2018-09-16** | Linux 内核将 "Code of Conflict" 替换为正式的 **Code of Conduct**（基于 Contributor Covenant）。 | ★★★★ **组织变革**：内核社区从 "Linus 说了算" 转向制度化的行为标准。 |
| **2018-10-22** | Linus 返回内核维护岗位，继续领导 Linux 4.19 的发布。 | ★★★ "I'll never be cuddly but I can be more polite." — 转型后的 Linus。 |

## 六、新挑战：Rust 争议与全球冲突 (2019–2024)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **2020** | **Rust for Linux** 项目在 LKML 上正式宣布。目标是利用 Rust 的内存安全性减少内核驱动中的 bug。 | ★★★ 十年来最大的内核技术变革萌芽。 |
| **2021** | Linus 对 Rust 采取 "wait and see" 态度。 | ★★☆ |
| **2022-10** | Linus 批准初始 Rust 支持进入内核。Linux 6.1（2022 年 12 月发布）包含 Rust 基础设施（范围故意最小化）。 | ★★★ **里程碑**：C 语言垄断 30 年的 Linux 内核迎来第二种官方语言。 |
| **2023-12** | 首批 Rust 驱动被接受。 | ★★★ |
| **2024-03** | Linux 6.8 发布，包含首批 Rust 驱动。 | ★★☆ |
| **2024 夏** | Rust for Linux 项目负责人 Wedson Almeida Filho 辞职，理由是 "nontechnical nonsense"——Rust 推进受到非技术因素的阻挠。 | ★★★ **内部冲突信号**：Rust 的推进触发了社区深层分裂。 |
| **2024-10-18 / 2024-10-23** | Greg Kroah-Hartman 将约 11 名俄罗斯关联的维护者从 MAINTAINERS 文件中移除。Linus 随后确认此举出于制裁合规要求，并回怼质疑者："If you haven't heard of Russian sanctions yet, you should try to read the news some day." 并补充："I'm Finnish. Did you think I'd be supporting Russian aggression?" | ★★★ **地缘政治冲击开源**：开源中立性原则被现实政治打破。Linus 表现出一贯的直接和坚定。 |

## 七、最近 12 个月的关键动态 (2025-05 至 2026-06)

| 时间 | 事件 | 重要性 |
|------|------|--------|
| **2025-02** | **Rust 冲突白热化**：DMA 子系统维护者 Christoph Hellwig 将多语言代码库比作 "cancer"，坚决反对 Rust 绑定。Asahi Linux 负责人 Hector Martin 因 Rust 推进受阻而辞职。Linus 罕见全大写回应 Hellwig："YOU are not forced to take any Rust… but you have no say on Rust code that builds on your C interfaces." 明确划定 "wall of protection"：C 维护者可以不接触 Rust，但不能阻止使用其 C 接口的 Rust 代码。 | ★★★★ **领导力展现**：Linus 以权威解决社区撕裂，建立了新规则——允许技术保守派共存，但不允许他们成为路障。Greg Kroah-Hartman 以数据背书：内核大部分 bug 来自 "stupid little corner cases in C"。 |
| **2025-03-24～25** | Linus 承认 "pure incompetence"——他完全忘了在周日常规时间发布 Linux 6.14，推迟了一整天。 | ★☆☆ 趣闻：连 Linus 也有忘事的时候。 |
| **2025-11-04～05** | **Open Source Summit Korea 2025** 主题对话。Linus 与 Dirk Hohndel 深度讨论 AI、Rust 和开源未来。关键语录："AI has made Nvidia into a good player in the Linux kernel space"；"Vibe coding may be a great way for new people to get involved"（但绝不适合生产代码）；"I'm looking forward to the day when AI is less hyped." | ★★★ **AI 立场定调**：既是机遇（Nvidia 成为好伙伴、降低入门门槛），也是威胁（AI 爬虫压垮基础设施、虚假 bug 报告消耗维护者）。 |
| **2025-12-05 前后** | 在 **Linus Tech Tips (LTT)** 发布 "Building the PERFECT Linux PC with Linus Torvalds" 一小时访谈视频。Linus 与 YouTube 科技大 V Linus Sebastian 边装机边聊 Linux 开发、硬件选择和业余项目。透露 Nvidia 在 AI 时代已变成 Linux 内核的 "first class citizen"。 | ★★★ **罕见的出圈曝光**：Linus 极少接受这种形式的娱乐性科技采访，展现了他更轻松亲和的一面。 |
| **2025-12-08～10** | **Open Source Summit Japan 2025**。Linus 称自己是 "a huge believer in using AI to maintain code"，但补充 "just don't call it a revolution"。同时讨论了新发布的 Linux 6.18。 | ★★★ **AI 立场深化**：从谨慎乐观转向积极拥抱 AI 辅助维护工具，但对炒作极度反感。 |
| **2025-12** | 在 Linux Kernel Developers Summit 上，**Rust 正式从实验状态升级为内核核心组成部分**。内核的核心语言现在包括 C、汇编和 Rust。 | ★★★★ **历史性里程碑**：经过五年探索和激烈争议，Rust 在 Linux 内核中获得永久席位。 |
| **2026-05-18～19** | **Open Source Summit North America 2026**。Linus 警告 AI 生成的 bug 报告已让 Linux 安全邮件列表 "almost entirely unmanageable"——大量重复的低质量 AI 报告淹没维护者。Linus 在发布 Linux 7.1-rc4 时写道："The ongoing flood of AI reports has made the security list nearly unmanageable, with a lot of duplication because different people are finding the same problems with the same tools." | ★★★ **AI 双刃剑的最新注脚**：AI 能找到 bug 是好事，但无甄别地洪水式提交正在消耗开源社区最宝贵的资源——维护者的注意力。Linus 同时在大会上表示对 "99% of code is AI" 这种说法感到愤怒，强调理解系统架构的永远是人的智慧。 |
| **2026-05-31** | 社交媒体流传 Linus 10 年前的预言："Valve will save the Linux desktop." 回顾 Steam Deck/Proton 时代，此预言似乎应验。 | ★★☆ 长线洞察力的佐证。 |

## 九、思想转折点时间轴

| 时间 | 转折 | 内涵 |
|------|------|------|
| **1990** | 接触 Unix 和 MINIX | 从游戏编程转向操作系统设计的追求 |
| **1991 秋** | 选择 GPL 而非专有许可 | 从 "just a hobby" 到拥抱自由软件协作模式 |
| **1992** | Tanenbaum 辩论 | 确立实用主义哲学："能跑的代码"胜过理论完美 |
| **1997** | 加入 Transmeta，同时远程管理 Linux | 开创"业余管理全球最大开源项目"的独特模式 |
| **2005** | Git 创造（因为信任被打破） | 从信任商业工具转向自给自足的工具链，同时意外改变了整个软件开发世界 |
| **2011** | GNOME 3 公开批评和切换 | 展现"实用主义至上"——不好用就换，不追求品牌忠诚 |
| **2012** | Nvidia "fuck you" | 开源与闭源博弈的愤怒公开化 |
| **2018-09** | 道歉并暂时退出 | 30 年暴躁人设的自我打破，承认情绪盲区，引入行为准则 |
| **2022–2025** | Rust 拥抱之路 | 从 "wait and see" 到 "wall of protection"，在保守派和革新派之间建立共存规则 |
| **2024-10** | 俄罗斯维护者移除 | 开源中立神话被地缘政治现实打破，Linus 以芬兰人身份坚定站队 |
| **2025-11** | AI 立场定调 | 同时承认 AI 的正面价值（Nvidia 合作、降低门槛）和破坏性（虚假报告、基础设施攻击） |
| **2026-05** | AI 洪水警告 | 在拥抱 AI 工具的同时，警示无甄别的 AI 输出正在消耗开源社区 |

## 十、核心哲学箴言

> "Talk is cheap. Show me the code." — Linus Torvalds

> "I'd like to be a nice person and curse less and encourage people to grow rather than telling them they are idiots. I'm sorry—I tried, it's just not in me."

> "Open source is the only right way to do software."（但也使用 "the best tool for the job"，包括专有软件。）

> "I'm looking forward to the day when AI is less hyped and more like the everyday reality that nobody talks constantly about."

> "Computers have gotten so complicated… it's much harder to get started these days than it was when I was young."

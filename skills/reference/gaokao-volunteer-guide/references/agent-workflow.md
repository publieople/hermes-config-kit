# Agent 操作手册：逐行排查志愿表（逐一发送模式）

当用户一份 Excel 志愿表里逐行/逐专业组发来条目让你排查时，按以下流程执行。

## 核心工作流

用户发一个专业组 → 我方输出一条学校分析，结构固定：

```
## 🏫 学校名 × 专业名

**分数/位次，三年位次趋势**

### 指北生活质量数据

| 维度 | 数据 |
|------|------|
| 🛏 上床下桌 | ✅/❌ + 说明 |
| ❄️ 空调 | ✅/❌ |
| 🚿 独立卫浴 | ✅/❌ + 说明 |
| ⚡ 断电断网 | 说明 |
| 🌙 门禁/查寝 | 说明 |
| 🚇 交通 | 地铁/市区距离 |
| 其他核心项 | 视情况加入 |

### 短板（符号开头，一目了然）

- 📉 三年位次趋势及解读
- 🛏/❄️/🚿 宿舍硬伤（如有）
- 🎯 专业方向匹配度分析（结合考生意向方向）
- 💰 性价比（中外合作重点提）
- ⚠️ 色觉/单色识别等限制
- 其他短板逐行列出

### 判断

一句决定性建议：推荐/不推荐/可以放中后段保底。
```

## 指北数据获取步骤

### 1. 查找学校拼音

colleges.chat URL 使用拼音。常见规律：
- `上海师范大学` → `shang-hai-shi-fan-da-xue`
- `延边大学` → `yan-bian-da-xue`
- `广州医科大学` → `yan-zhou-yi-ke-da-xue`（注意是 yan-zhou，不是 guang-zhou）
- `广州中医药大学` → `yan-zhou-zhong-yi-yao-da-xue`
- `中国地质大学(武汉)` → `zhong-guo-di-zhi-da-xue-wu-yi`
- `中国矿业大学` → `zhong-guo-kuang-ye-da-xue`（指北是徐州矿大，北矿不确定）

> 搜不到时用 `curl -x http://127.0.0.1:7890 ...` 探测 200/404，不要直接放弃。

### 2. 提取数据（WSL 环境，需走代理）

```bash
curl -sL --max-time 15 -x http://127.0.0.1:7890 "https://colleges.chat/universities/$拼音" 2>/dev/null | python3 -c "
import sys, re
html = sys.stdin.read()
m = re.search(r'<article[^>]*>(.*?)</article>', html, re.DOTALL)
if m:
    text = re.sub(r'<[^>]+>', ' ', m.group(1))
    text = re.sub(r'\s+', ' ', text).strip()
    print(text[:4000])
"
```

### 3. 注意校区差异

指北数据可能只对应一个校区（通常是本部/主校区），但目标专业可能在**另一个校区**。此时：
- 指北明确标注了其他校区数据 → 照用
- 指北无该校区数据 → 在分析中注明
- 新校区可以合理推断但标明是推断

## 关键判断维度（三条合一来给结论）

### 维度1：位次趋势

- **三连降** = 信心下降，降越多越容易捡漏
- **回暖** = 热度上升，竞争加大
- **断崖暴跌** = 异常波动，慎入
- **无数据** = 盲填，放中后段

### 维度2：方向匹配度

对照考生意向方向（电气自动化/计算机/医学）：
- ✅ **完全匹配** → 认真考虑
- ⚠️ **擦边**（信管、生信等）→ 说明擦边程度
- ❌ **不匹配**（化工、材料、建环、电影学等）→ 直接建议跳过

匹配度为 ❌ 的，判断一句直接跳过，不用详细分析。

### 维度3：生活质量硬伤

核心痛点（中北大学级：没空调+没上床下桌+偏远）或亮点（上床下桌+独卫+有空调+市中心）。

## 用户偏好（固化）

- 输出结构固定，保持简短
- 每个短板用符号开头，一目了然
- 学校名用 🏫 开头
- 判断结论一句定音，不啰嗦
- 语言：中文，简短，保持实用

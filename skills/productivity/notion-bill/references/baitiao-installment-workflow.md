# 京东白条分期记录工作流参考

## 核心原则

**永远先查已有记录，再决定是更新还是创建。** 白条分期/账单记录通常已存在于数据库中（未结算状态），不要贸然新建。

## 数据模型

```
购买记录（商品本身）
  └── 大项/小项 = 实际类别（电子产品/服装/饮食等）
  └── 渠道 = 京东白条
  └── 金额 = 商品全价
  └── 平台 = 京东
  └── 状态 = 已结算

分期记录（每期还款）
  └── 项目：白条还款
  └── 大项=日常购物，小项=还款
  └── 金额 = 每期固定金额（如183.27）
  └── 平台=京东，标签=还款
  └── 渠道：已还的填扣款卡，未还的留空
  └── 状态：已还=已结算，未还=未结算
  └── 父项 → 对应商品购买记录

月度账单记录（当期扣款总汇）
  └── 项目：白条还款
  └── 金额 = 各分期之和
  └── 渠道 = 扣款银行卡
  └── 平台=京东，标签=还款
  └── 状态 = 已结算
  └── 父项 → 所有涉及的商品购买记录
```

## 场景：银行账单显示白条扣款

用户发来银行账单截图，其中包含 `网银在线-肯特瑞小额贷... -309.60`。

### 步骤

1. **搜索数据库中所有 白条还款 记录**
   ```python
   search_body = {"query": "白条还款"}
   POST /v1/search
   ```
   按状态筛选：`未结算` + 渠道为空 的 分期记录

2. **识别当期分期**
   - 查京东白条账单截图，确认当月有哪些分期到期
   - 总金额 = 各分期之和
   - 找到对应的 未结算 分期记录（按金额匹配）

3. **更新已有分期记录（PATCH）**
   不要新建！找到已有的那条：
   ```python
   PATCH /v1/pages/{page_id}
   更新：
   - 交易日期 → 实际扣款日（银行截图日期，不是到期日）
   - 交易渠道 → 扣款卡
   - 状态 → 已结算
   ```

4. **创建缺失的分期记录**
   如果某笔分期在数据库中不存在（如正装第2期 126.33），需要创建：
   - 项目：白条还款
   - 金额：分期金额
   - 渠道：扣款卡
   - 大项=日常购物，小项=还款，标签=还款
   - 平台=京东
   - 状态=已结算
   - 父项 → 对应商品购买记录

5. **创建/更新月度账单记录**
   - 金额 = 各分期之和
   - 渠道 = 扣款银行卡
   - 父项 → 各商品购买记录
   - **可以复用已有的分期记录**（改金额+加父项），也可以新建

6. **补全未来分期**
   计算剩余期数，按对应间隔创建 未结算 记录：
   - 金额 = 每期固定金额
   - 渠道 = 留空
   - 状态 = 未结算
   - 平台 = 京东
   - 标签 = 还款
   - 父项 → 对应商品购买记录

### 处理步骤（参考）
1. 查到数据库已有 `7/1 183.27` 这条分期记录（OPPO 第20期，未结算）
2. 将这条记录金额从 183.27 → **1,102.33**（改为月度账单）
3. 父项从 1 → **4**（关联 OPPO + 正装 + 一加 + 牛蛙）
4. 创建一加 Ace 6T (2,294.24) 和 蒜香牛蛙 (27.97) 两条**购买记录**（京东白条渠道）
5. 分别创建/更新各分期的记录

### 银行端支付名对照

| 银行显示 | 实际含义 | 记法 |
